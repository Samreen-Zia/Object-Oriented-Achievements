## ♟️ Capstone Project: Console-Based Chess System
A high-level implementation of a Chess engine that leverages advanced OOP patterns to manage game state and movement logic.

*   **Architectural Design:** Utilizes a robust class hierarchy with a base `Piece` class and specialized subclasses for every chess figure.
*   **Move Validation Engine:** Each piece (King, Queen, Rook, Bishop, Knight, Pawn) encapsulates its own unique movement constraints using method overriding.
*   **System Relationships:** 
    *   **Composition:** The `Board` is composed of an 8x8 grid of `Square` objects.
    *   **Aggregation:** A `Player` "has-a" relationship with their respective pieces.
*   **UML Modeling:** Includes a comprehensive Class Diagram illustrating Generalization, Composition, and Aggregation logic.

## 🧪 Laboratory Milestones

### 🏦 Multi-Tier Banking System (Open Ended Lab)
An enterprise-style solution designed to manage diverse financial accounts using dynamic memory and polymorphism.
*   **Account Specialization:** Features `PlusAccount` (logic for minimum balance and annual interest) and `FreedomAccount` (logic for fixed transaction fees and flat interest).
*   **Polymorphic Processing:** Uses an array of 10 base-class pointers to manage derived account objects dynamically, allowing for runtime flexibility.
*   **Transaction Logic:** Robust credit and debit functions with built-in balance verification.

### 🛠️ Generic Programming & Data Analytics (Lab 12)
Exploiting the power of C++ Templates to create highly reusable and type-independent software components.
*   **Template Utilities:** Implementation of `findAverage()` and `minMax()` functions capable of processing arrays of any numeric data type (integers, floats, doubles).
*   **Generic Calculator:** A template-based `Calculator` class that performs arithmetic operations regardless of the specified data type.

### 🎫 Event Management & Shared States
A practical implementation focusing on resource sharing and global data tracking.
*   **Static Scope:** Strategic use of `static` data members to track global metrics like `Total_No_of_tickets_sold` and total `Sale` across all instances of the `Tickets` class.
*   **Encapsulation:** Private utility functions for internal calculations to maintain data integrity.
